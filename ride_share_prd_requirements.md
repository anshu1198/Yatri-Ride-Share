# Ride Share Project Requirements

## Core Concept
A full-stack MERN application for a Ride Share service.

## Key Features & Flows
1. **Authentication & Security**
   - User Registration & Login
   - JWT-based authentication
   - Password hashing with bcrypt
   - Protected routes

2. **Core Functionality (CRUD)**
   - **Create**: Add new ride requests or offers
   - **Read**: View available rides on a dashboard
   - **Update**: Modify ride details or user profile
   - **Delete**: Cancel or remove rides

3. **Technical Stack**
   - **Frontend**: React (Vite), Tailwind CSS, React Router, Axios, Lucide React Icons, React Toastify
   - **Backend**: Node.js, Express.js, MongoDB (Mongoose)
   - **Authentication**: JWT, bcryptjs

4. **UI/UX Requirements**
   - Clean, responsive mobile interface
   - Modern design system (likely using a professional, trust-inspiring palette like deep blues and whites)
   - Clear navigation and status feedback

## Planned Screens
1. **Welcome/Onboarding**: High-impact introduction to the app.
2. **Login/Register**: Forms for user authentication.
3. **Dashboard/Home**: Main feed showing available rides, search/filter, and a "Create Ride" CTA.
4. **Ride Details**: Detailed view of a specific ride with booking/contact options.
5. **User Profile/My Rides**: Managing personal info and active/past trips.
